# Earguard 앱 기획서 — 00. 프로젝트 개요 및 사이트맵

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | Earguard |
| 개발 유형 | 모바일 앱 (iOS / Android) |
| 앱 성격 | 청각 건강 관리 + 소음 측정 + 마음 건강 케어 |
| 슬로건 | "당신의 동네는 얼마나 조용한가요?" |
| 참조 디자인 | ODSO 제공 Figma (AI 생성 화면 기반) |

### 앱 핵심 목적
- 이어폰/주변 소음으로부터 사용자의 청각 건강을 모니터링하고 관리
- 일일 소음 할당량을 기준으로 과도한 소음 노출 경고
- 청력 검사 및 마음 건강 체크를 통한 종합 건강 관리
- 소음 커뮤니티를 통해 조용한 경로·장소 정보 공유

---

## 2. 연동 기기

- **Eartag 기기 (하드웨어)**: 블루투스 연결을 통해 소음 데이터 측정
  - 기기 미보유 시에도 스마트폰 마이크로 앱 단독 사용 가능
- **애플 건강 앱 (Health Kit) 연동**: 소음 항목 및 건강 데이터 연동
  - ⚠️ iOS Apple Health의 환경 소음 항목과 연동 가능 여부 사전 확인 필요
  - Android의 경우 Google Health Connect 연동 검토 필요

## 3. 전체 사이트맵

```
Earguard
│
├── 온보딩 (최초 1회)
│   ├── Step 1 — 거주지 설정
│   ├── Step 2 — 청각 베이스라인 구축
│   │   ├── 2-1. 기저질환 선택
│   │   ├── 2-2. 순음 청력 검사
│   │   ├── 2-3. 소음하 청취 능력 검사
│   │   └── 2-4. 검사 결과
│   ├── Step 3 — 마음 건강 체크
│   └── Step 4 — 기기 연동 및 권한 설정 (Eartag 연결 / 건강 데이터 / 위치 / 마이크)
│
├── [탭 1] 홈
│   ├── 홈 메인 (실시간 소음 + 일일 할당량 + 소음 지도 + 청력 나이 + 챌린지 스트릭 + 미니플레이어)
│   └── 소음 안전 지도 상세보기 (팝업 — 위치별 소음 현황)
│
├── [탭 2] 기록
│   ├── 소음 할당량 탭
│   ├── 위치 기록 탭
│   │   └── 장소 상세 (바텀시트)
│   ├── 행동 분석 탭 (AI)
│   └── 정밀 자기검사 탭
│       ├── 순음 청력 검사
│       ├── 소음의 청취 능력
│       ├── 소음 노출 설문
│       └── 청력 건강 설문
│
├── [탭 3] 마음챙김
│   ├── [힐링 탭] 메인 (기분 기록 + AI 스트레스 분석 + AI 힐링 가이드 + 힐링 사운드 라이브러리)
│   │   ├── AI 스트레스 분석 (상세 팝업)
│   │   └── AI 힐링 가이드 (팝업)
│   └── [기분 기록 탭] 오늘 기분 체크 + 캘린더 형식 기분 이력 조회
│
├── [탭 4] 챌린지
│   ├── 메인 (레벨 + 오늘의 미션)
│   ├── 미션 목록 (스크롤)
│   └── 업적 + 보상 상점
│
├── [탭 5] 커뮤니티
│   ├── 지도 탭 (소음 핀 지도)
│   ├── 목록 탭
│   │   └── 필터
│   ├── 소음 제보하기 (팝업)
│   └── 조용한 출근길 찾기
│       └── 경로 추적 중 화면
│
└── [탭 6] 설정
    ├── 설정 메인
    ├── 프로필 수정
    ├── 비밀번호 변경
    ├── 건강 정보 수정
    └── 기기 설정
```

## 4. 주요 데이터 흐름 요약

| 데이터 | 출처 | 상태 |
|---|---|---|
| 거주지 설정 행정구역 데이터 (시/도/구/군/동 목록) | ODSO 제공 | ⏳ 전달 예정 |
| 마음 건강 체크 설문 항목 | ODSO 제공 | ⏳ 전달 예정 |
| 일일 소음 할당량 계산식 | ODSO 제공 (안전범위 100 기준 누적 계산식) | ⏳ 전달 예정 |
| 청력 나이 계산식 | ODSO 제공 | ⏳ 전달 예정 |
| 청력검사 로직 | 모두의보청기 소스코드 | ⏳ 소스 전달 예정 |
| 마음챙김 힐링 사운드 콘텐츠 | 마인드톤 소리치료 콘텐츠 | ✅ 활용 확정 (전달 완료) |
| 챌린지 미션 내용 | 관리자 페이지에서 설정 | 🔧 어드민 기능 필요 |
| 청각 베이스라인 항목 | 관리자 페이지에서 수정 가능 | 🔧 어드민 기능 필요 |
| 실시간 소음 측정 | Earguard 기기 (BLE) 또는 스마트폰 마이크 | 개발 필요 |

## 5. 어드민(관리자) 페이지 관련 항목 요약

> 어드민 페이지 기획은 별도 진행 예정. 아래 항목은 어드민에서 관리 가능해야 함.

- 청각 베이스라인 구축 시 기저질환 항목 추가/수정/삭제

- 힐링 사운드 라이브러리 콘텐츠 관리 (마인드톤 소리치료 콘텐츠 넣기)
