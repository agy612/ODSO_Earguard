# Earguard 앱 기획서 — 02. 온보딩

---

> 온보딩은 회원가입(또는 소셜 로그인 최초 가입) 직후 1회만 진행.

---

## 2-1. Step 1 — 거주지 설정

> **참조 화면**: screen_04, screen_05

### 화면 목적
사용자 거주 지역 기반 소음 안전도 확인 및 맞춤 정보 제공을 위한 거주지 설정.

---

### 화면 레이아웃

```
┌─────────────────────────────────────────┐
│ [Step 1 ●○○○]                           │
│                                         │
│         [위치 아이콘]                    │
│         거주지 설정                      │
│    우리동네의 소음 안전도를 확인해보세요   │
│                                         │
│  시/도 [선택 ▾]                         │
│  동    [선택 ▾]                         │
│                                         │
│  [현재 위치로 설정]                      │
│                                         │
│  전국에서 소음 안전도가 상위 13%인       │
│  지역입니다                              │  ← 선택 후 표시
│                                         │
│  [이전 (비활성)]              [다음]     │
└─────────────────────────────────────────┘
```

---

### UI 구성
| 요소 | 설명 |
|---|---|
| 프로그레스 바 | Step 1 표시 |
| 위치 아이콘 | 초록색 핀 아이콘 |
| 타이틀 | "거주지 설정" |
| 서브 텍스트 | "우리동네의 소음 안전도를 확인해보세요" |
| 시/도 선택 드롭다운 | ODSO 제공 데이터 기준, 검색 기능 포함 |
| 동 선택 드롭다운 | 시/도 선택 후 연동 — 연동 방식은 ODSO 데이터 구조에 따라 결정, 검색 기능 포함 |
| 현재 위치로 설정 버튼 | GPS 권한 요청 후 자동 입력 |
| 소음 안전도 결과 표시 | 거주지 선택 후 해당 지역의 전국 소음 안전도 백분위 표시 (예: "전국에서 소음 안전도가 상위 13%인 지역입니다") |
| 다음 버튼 | 선택 완료 시 활성화 |
| 이전 버튼 | 비활성 (첫 단계) |

### 데이터 / 연동
- ⚠️ **시/도 · 동 선택 목록 데이터: ODSO 제공 예정**
  - 데이터 구조 및 형식 전달 후 드롭다운 연동 방식 확정 (검색 기능 포함)
  - 시/도 선택 시 동 목록 연동 여부도 전달받은 데이터 기준으로 결정

## 2-4. Step 4 — 기기 연동 및 권한 설정

> **참조 화면**: screen_11

### 화면 목적
Eartag 블루투스 기기 연결 및 앱 동작에 필요한 권한(건강 데이터, 위치 등) 일괄 설정.

---

### 화면 레이아웃

```
┌─────────────────────────────────────────┐
│ [Step 4 ○○○●]                           │
│                                         │
│         Eartag 연동                     │
│    기기를 소음 측정기로 연결하세요        │
│                                         │
│         [Eartag 기기 이미지]             │
│                                         │
│         [검색 중... / 기기 발견]         │
│                                         │
│  [연결하기]                              │
│  [기기 없이 계속하기]                    │
└─────────────────────────────────────────┘
```

---

### UI 구성
| 요소 | 설명 |
|---|---|
| 프로그레스 바 | Step 4 표시 |
| 타이틀 | "Eartag 연동" |
| 서브 텍스트 | "기기를 소음 측정기로 연결하세요" |
| 기기 이미지 | Eartag 기기 이미지 |
| 기기 상태 표시 | 검색 중 / 기기 발견 / 연결 완료 상태 |
| 연결하기 버튼 | BLE 스캔 및 연결 |
| 기기 없이 계속하기 | 앱 단독 사용으로 진행 (스마트폰 마이크 활용) |

### 권한 요청 항목
| 권한 | 용도 |
|---|---|
| 블루투스 | Eartag 기기 연결 및 소음 데이터 수신 |
| 위치 | 위치 기반 소음 기록, 조용한 경로 탐색 |
| 건강 데이터 앱 연동 | Apple Health(iOS) / Google Health Connect(Android) — 소음 항목 및 신체 데이터 연동 |
| 마이크 | 기기 미연결 시 스마트폰 마이크로 소음 측정 |

### 기능 정의
- 블루투스 권한 요청 → BLE 스캔 → 근처 Eartag 탐색 → 선택 후 연결
- 건강 데이터 앱 연동 권한 요청 (거부 시 설정 > 건강 정보에서 추후 연동 가능)
- 위치 권한 요청 (거부 시 위치 기반 기능 제한 안내)
- 마이크 권한 요청
- 기기 없이 계속하기 → 스마트폰 마이크로 소음 측정, 홈으로 이동
- 연동된 기기 정보는 설정 > 기기 설정에서 관리

## 온보딩 전체 플로우 요약

```
회원가입 / 소셜 로그인
    ↓
Step 1. 거주지 설정 (ODSO 데이터 연동 예정)
    ↓
Step 2. 청각 베이스라인 구축
    ├─ 2-1. 기저질환 선택 (어드민 항목 관리)
    ├─ 2-2. 순음 청력 검사 (모두의보청기 소스 활용)
    ├─ 2-3. 소음의 청취 능력 (모두의보청기 소스 활용)
    └─ 2-4. 검사 결과 (청력 나이 계산식 ODSO 제공 예정)
    ↓
Step 3. 마음 건강 체크 (설문 항목 ODSO 제공 예정)
    ↓
Step 4. Eartag 기기 연동 (선택, Skip 가능)
    ↓
홈 화면
```
